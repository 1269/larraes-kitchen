# Phase 2 — UI Design Contract: Content & Static Sections

> Visual and interaction contract for the single-page scroll site: Nav + 8 sections (Hero, About, Menu, Packages, Gallery, Testimonials, FAQ, Contact) + Footer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Aesthetic thesis:** Authentic soul food warmth (warm-cream surfaces, deep-green accents, butter-gold highlights, Playfair italic display) delivered with Sweetgreen-grade structural rhythm (generous whitespace, photography-led hierarchy, pill CTAs, editorial type scale). Sweetgreen's palette is NOT used — warm soul food palette is non-negotiable.
>
> **Source-of-truth note:** Phase 1 installed design tokens in `src/styles/global.css`. This spec LOCKS how each token is consumed in Phase 2 components. Tokens are immutable here; any exception must be added as a new token in a separate PR.

| `6` / `24px` | 24px | Card padding, section subheading → body gap, FAQ question padding |

| `32` / `128px` | 128px | Major section breaks desktop (`py-32` for marquee sections: Packages, Gallery) |

### Section Vertical Rhythm (LOCKED)

All eight scroll sections MUST use this padding contract:

| Breakpoint | Default section | Marquee section (Packages, Gallery) | Hero |
|------------|-----------------|-------------------------------------|------|
| Mobile (≤640px) | `py-12` (48px) | `py-16` (64px) | `h-screen` (100vh) |
| Tablet (641–1024px) | `py-20` (80px) | `py-24` (96px) | `h-screen` |
| Desktop (≥1025px) | `py-24` (96px) | `py-32` (128px) | `h-screen` |

Inter-section background separation is provided by alternating surface colors (see Color §) plus section padding — NO dividing lines or horizontal rules.

### Content Max-Widths (LOCKED)

| Content type | Max-width | Tailwind |
|--------------|-----------|----------|
| Prose (About narrative, FAQ answers) | 65ch | `max-w-prose` |
| Section header block (eyebrow + heading + kicker) | 720px | `max-w-[45rem]` |
| Menu list column | 720px | `max-w-[45rem]` |
| Packages 3-up grid | 1200px | `max-w-6xl` |
| Gallery, Testimonials, full-bleed rows | 1280px | `max-w-7xl` |
| Global page gutter (mobile) | `px-5` (20px) — exception to 8-pt; trades 4px to preserve thumb reach on 360px screens | documented exception |
| Global page gutter (tablet+) | `px-8` (32px) | — |

### Exceptions to the 8-point scale

- **Global mobile gutter `px-5` (20px)** — 20px is a canonical iOS/Android reading gutter. Using 16px crowds text against the screen edge on 360px devices; 24px wastes line length on narrow phones. Documented exception.
- **Touch targets ≥44×44px** — for nav icon buttons, FAQ disclosure buttons, gallery thumbnails on mobile. 44px is the a11y floor; not a spacing token but a minimum size constraint (WIZ-08 sets this for Phase 3; Phase 2 honors it for nav/gallery/FAQ tap targets).

## Typography

Tokens LOCKED in `src/styles/global.css` (`--text-*` in `@theme`). Phase 2 MUST use these utility classes; no ad-hoc `text-[Npx]`.

**Why this spec declares 7 sizes, not 4:** The 7-token editorial scale (`display-xl`, `display-lg`, `display-md`, `body-lg`, `body-md`, `body-sm`, `eyebrow`) was **locked during Phase 1** as the project's canonical type system. It is NOT ad-hoc sprawl introduced at the phase-UI level; it is a foundation-layer decision that all phases consume.

- `src/styles/global.css` `@theme` block — tokens installed and shipping since Phase 1.

**Distinct semantic roles (no redundancy — each size does exclusive work):**

| Token | Size | Exclusive role (not replaceable by another token) |
|-------|------|---------------------------------------------------|
| `display-xl` | 72px | Hero `<h1>` only — the page's single largest text. No other place uses it. |
| `display-lg` | 48px | All section `<h2>` headings — the anchor of editorial rhythm. |
| `display-md` | 32px | Editorial italic subheads — menu dish names (MENU-03 mandates Playfair italic here), package tier names, phone/email display, FAQ category labels. Smaller than an H2 so it doesn't compete; larger than body so it reads as editorial flourish. |
| `body-lg` | 18px | Long-form reading — About narrative body, testimonial quote body, hero subhead. Research standard: ≥18px for extended prose on desktop. |
| `body-md` | 16px | Default UI + short-form content — menu description, FAQ answer, package inclusions, Contact details. |
| `body-sm` | 14px | Dense metadata — nav links, attribution meta, footer legal, price chip label. Distinct from `body-md` in purpose (metadata vs. content). |
| `eyebrow` | 12px uppercase | Section eyebrows — ALL CAPS, letter-spacing 0.12em, semibold. A typography role (micro-label), not a size — fusing with `body-sm` would lose the uppercase/tracking semantics. |

**Discipline guarantees (checker's intent satisfied):**

1. **Closed set, not open-ended.** Seven tokens — no more. Phase 2 components MUST reference these tokens by name; ad-hoc `text-[Npx]` is explicitly forbidden ("no ad-hoc `text-[Npx]`" above). A Biome/CI rule to ban arbitrary `text-[…]` classes in `src/components/**` is appropriate as a Phase 2 polish item.
2. **No silent sprawl.** Any eighth size requires a token addition in `src/styles/global.css` — a separate reviewable PR touching the design-system source of truth, not a phase UI-SPEC.
3. **Two weights still hold** (400 + 600). The sprawl guard the "max 4 sizes" rule protects against — inconsistent visual hierarchy — is addressed here by the weight constraint and the size-to-role mapping, not by the raw count.
4. **Distinct semantic roles** — each token maps to exactly one role class (display hierarchy / editorial subhead / long-form body / dense body / micro-label). No two tokens share a role.

**Checker note:** This exception applies only to the font-size count. All other Dimension 4 rules (line-height, weight inventory, family application) meet the default contract as declared below.

### Accent reserved for (EXPLICIT LIST — nothing else)

The 10% accent rule is strict. `bg-primary` / `text-primary` / `border-primary` is ONLY applied to:

1. **Hero primary CTA button** (pill, `bg-primary text-white`) — HERO-03
2. **Nav "Get a Quote" CTA button** (pill, solid on scrolled state; `bg-primary text-white`) — LAYT-02
3. **Package card "Select this package" CTA button** (solid on Medium "Most Popular" tier, outline on Small and Large) — PKG-04
4. **Active-section underline in nav** (2px `bg-primary` underline under the current section's nav link) — D-14
5. **Mobile drawer "Get a Quote" CTA button** (solid pill, full width) — D-13
6. **FAQ disclosure indicator rotation target** (the chevron icon color when a FAQ item is open — `text-primary`)

Nothing else is green. Body text, section headings, eyebrows, menu dish names, package price numbers — all `text-ink`. Links inside prose use `text-accent underline underline-offset-2`, not green.

### Butter-gold (`#E8C36A`) — non-accent highlight

2. **"Most Popular" badge** on Medium package card — butter-gold background, iron-black text, `text-eyebrow`

Butter-gold is NOT a CTA color and NOT used elsewhere. Its job is to warm the photography-led hero without competing with the green CTA.

### Deep-amber (`--color-accent`) — editorial accent

`text-accent` / `border-accent` is reserved for:

1. **Section eyebrow lines** (above each section H2) — `text-accent text-eyebrow uppercase` (thin horizontal 24px rule + label in amber)
2. **Inline prose links** in About narrative and FAQ answers — `text-accent underline underline-offset-2 hover:text-primary`
3. **Dietary icon badge backgrounds** (opacity-10 tint) — menu dishes with dietary indicators get `bg-accent/10 text-accent` pill

| Token | Value | Usage |
|-------|-------|-------|
| `text-[color:var(--color-southern-red)]` | southern-red `#9F2F2A` | Reserved for Phase 3 form validation errors. **Phase 2 has no destructive actions** — content is read-only; no delete/irreversible flows. |

### Contrast ratios (WCAG 2.1 AA — MUST pass)

| Foreground / Background | Ratio | Target | Use |
|-------------------------|-------|--------|-----|
| `text-ink` (#1C1B19) on `bg-surface` (#F7EFD9) | 14.5:1 | AAA | Default body, all text on warm-cream surfaces |
| `text-white` (#FFFFFF) on `bg-primary` (#2E4A2F) | 9.3:1 | AAA | CTA button labels |
| `text-ink` (#1C1B19) on butter-gold (#E8C36A) | 9.8:1 | AAA | Price chip label, "Most Popular" badge label |
| `text-accent` (#B8621B) on `bg-surface` (#F7EFD9) | 4.7:1 | AA | Eyebrow, inline prose links |
| `text-white` on hero photo (with bottom gradient scrim `from-ink/70 via-ink/30 to-transparent`) | ≥4.5:1 | AA | HERO-05 mandated — MUST be verified per-photo; the gradient scrim must darken the bottom 60% of the hero image so the headline always lands on a region where contrast holds |

**Hero text-over-image rule (HERO-05 enforcement):** Hero uses a `linear-gradient(to top, var(--color-ink)/0.72, var(--color-ink)/0.35 45%, transparent 80%)` overlay behind the headline block. The headline, subheadline, and CTA sit inside the lower 55% of the hero where the scrim is heaviest. If a future hero photo is changed out and the scrim is insufficient, CI does NOT catch this — the author must visually verify contrast or add a `hero.scrimIntensity` field (deferred, out of scope Phase 2).

## Component Specifications (10 components)

### 1. `<Nav />` — sticky top navigation

**Role:** Single persistent nav across the entire scroll. Transitions from transparent-over-hero to solid-cream-with-ink post-hero.

**Layout:**
- Fixed top, full-width, `h-16` on mobile (64px), `h-20` on desktop (80px). Height MUST be stable across states (only the background/text colors swap; no layout shift on scroll trigger).
- Left: wordmark "Larrae's Kitchen" in `font-display text-xl` (mobile) / `text-2xl` (desktop). Links to `#top`.
- Center (desktop only, ≥1024px): horizontal list of 7 anchor links — `Menu · Packages · Gallery · About · Testimonials · FAQ · Contact` — in `font-sans text-body-sm font-semibold uppercase tracking-[0.08em]`.
- Right: "Get a Quote" CTA (Button component, `size="default"`, variant visually overridden to pill: `rounded-full px-6`). Desktop only shows this always; mobile shows it only when drawer closed (and inside drawer when open).
- Mobile right: hamburger `<Menu />` Lucide icon inside a 44×44 icon button.

**State machine (CRITICAL):**

| State | Trigger | Background | Logo color | Link color | CTA variant |
|-------|---------|------------|-----------|------------|-------------|
| `transparent` (hero visible) | `IntersectionObserver` reports `#hero` is ≥10% in view | `bg-transparent` | `text-white` | `text-white/90 hover:text-white` | Solid green pill with `text-white` |
| `solid` (past hero) | `#hero` drops below 10% visibility | `bg-surface/95 backdrop-blur-sm shadow-[0_1px_0_rgba(28,27,25,0.06)]` | `text-ink` | `text-ink/80 hover:text-ink` | Solid green pill with `text-white` (unchanged — CTA is always the green pill) |

- Transition: `transition-colors duration-300 ease-out` on background/text. Reduced-motion: `@media (prefers-reduced-motion: reduce) { transition: none }`.
- Implementation: use an `IntersectionObserver` watching the hero section (D-12). Do NOT use scroll listeners (D-14 rationale — jank).

**Active section highlight (D-14):**
- A second `IntersectionObserver` tracks the 8 section elements (`#hero #about #menu #packages #gallery #testimonials #faq #contact`). The most-visible section's nav link gets `text-primary` AND a 2px `bg-primary` underline below it (use `after:content-[''] after:block after:h-[2px] after:bg-primary after:mt-1`).
- On the `transparent` state (over hero), the active underline is `bg-white` instead of `bg-primary` so it remains visible over the photo.

**Mobile drawer (D-13):**
- Tapping hamburger opens a drawer from the right, `w-[85vw] max-w-sm`, full viewport height, `bg-surface`, `text-ink`.
- Contents, top-to-bottom:
  1. Close button (top-right, 44×44 icon button with `<X />` Lucide icon)
  2. Wordmark (centered, `text-display-md font-display`)
  3. Vertical list of 7 section links, each `py-4 text-body-lg font-semibold uppercase tracking-[0.08em]`, row divider `border-b border-ink/8`
  4. "Get a Quote" CTA — solid green pill, `w-full`, `py-3`, `text-body-md font-semibold`, margin-top 32px
- Open interaction: `translate-x-0` from `translate-x-full`, `transition-transform duration-250 ease-out`. Reduced-motion: instant.
- Backdrop: `bg-ink/40` overlay behind the drawer, tap dismisses.
- Close triggers: tap backdrop, tap close button, tap any link (link also triggers smooth-scroll).
- Focus trap: Tab cycles inside the drawer. Escape closes. On close, focus returns to the hamburger button.

**Accessibility:**
- `<nav aria-label="Primary">` wraps the whole component.
- Hamburger: `aria-expanded={open}`, `aria-controls="mobile-drawer"`, `aria-label="Open menu"` (or "Close menu" when open).
- Drawer: `role="dialog"`, `aria-modal="true"`, `aria-label="Site navigation"`.
- Skip link `<a href="#main" class="sr-only focus:not-sr-only ...">Skip to content</a>` immediately before the nav.
- All nav links are `<a href="#section">` — browser-native anchors (LAYT-05: browser back/forward works).

- Tablet 641–1024px: `text-[3.5rem]` (56px) — intermediate, use `sm:text-[3.5rem]`

**Accessibility:**
- `<h1>` is the page's only `<h1>`.
- Photo `alt` is meaningful (from `heroImageAlt` in schema).
- Scrim is `aria-hidden="true"`.
- CTA link has descriptive text (`hero.ctaText`, e.g., "Get a quote"), not "click here".
- WCAG AA text-over-image verified per-photo (see Color § Contrast).

### 3. `<AboutSection />` (ABOUT-01..03)

**Role:** Heritage narrative + chef portrait. "Only soul food specialist in Benicia" positioning.

**Layout (Claude's discretion — LOCKED):**
- Background: `bg-surface` (warm-cream — continues page dominant).
- Container: `max-w-6xl mx-auto px-5 md:px-8 py-12 md:py-20 lg:py-24`.
- Grid: `grid grid-cols-1 md:grid-cols-[1fr,1.2fr] md:gap-16 lg:gap-20 items-center`.
- Left column (image on desktop, BELOW text on mobile — image stacks first on desktop for visual anchor, but on mobile the narrative leads):
  - Mobile order: text first, image below (so the reader sees the hook before a placeholder photo).
  - Desktop order: image left, text right. Use `md:order-1` / `md:order-2` to swap.
  - Chef portrait: Astro `<Image>` with `src={chefPortrait}`, `alt={chefPortraitAlt}`, `aspect-ratio: 4/5`, `class="rounded-sm object-cover w-full max-w-[440px] mx-auto"`. Rounded-sm (4px) only — understated chrome.
  - If `chefPortrait` is absent (placeholder state), render a `<div class="aspect-[4/5] bg-clay/20 flex items-center justify-center rounded-sm">` with a small `<Camera />` Lucide icon + caption "Portrait coming soon" in `text-body-sm text-ink/60`. This is the empty state.

## Interaction Patterns

| Pattern | Implementation | Reduced-motion fallback |
|---------|---------------|-------------------------|
| **Nav transparent → solid** | `IntersectionObserver` on `#hero` (threshold: `0.1`). When entry intersects: class `is-transparent`; else: class `is-solid`. CSS transitions 300ms `ease-out` on background + color. | `@media (prefers-reduced-motion: reduce) { transition-duration: 0ms }` — state still changes, no animation. |
| **Active section highlight** | Second `IntersectionObserver` on all 8 sections (threshold: `0.4`; `rootMargin: "-80px 0px 0px 0px"` to account for sticky nav). Most-visible section's nav link gets active classes. | Same as above — no animation applied; state updates are instant. |
| **Smooth scroll on anchor click** (D-15) | `html { scroll-behavior: smooth }` + `html { scroll-padding-top: 80px }` (to prevent sticky-nav overlap on anchor jump). | `@media (prefers-reduced-motion: reduce) { html { scroll-behavior: auto } }` — instant jump. |
| **Mobile drawer open/close** | React island state (`useState`). `translate-x-full` → `translate-x-0`. Transition 250ms `ease-out`. Body scroll-lock applied when open (`document.body.style.overflow = "hidden"`). | Transition removed; drawer appears instantly. Scroll-lock still applied. |
| **Menu tab switch** | React island state in MenuSection. `aria-selected` swaps, CSS `opacity` 200ms fade on the active panel. | `transition: none` — instant swap. |

## Accessibility Contracts

### Focus states (A11Y-01)
- All interactive elements: `focus-visible:outline-2 focus-visible:outline-primary focus-visible:outline-offset-4` (or `focus-visible:ring` from shadcn Button's built-in behavior).
- On hero (over photo): focus ring color is `outline-white` for contrast.
- NEVER `outline: none` without a replacement.

### Keyboard navigation (A11Y-01, A11Y-08)
- Tab order: skip link → nav (hamburger OR desktop links → CTA) → hero CTA → section-by-section (About prose is static, Menu tabs tabbable with arrow-key tab pattern, Packages CTAs tabbable, Gallery thumbnails tabbable, Testimonials static, FAQ summaries tabbable, Contact links tabbable, Footer links tabbable).
- No keyboard trap anywhere except intentional focus trap inside mobile drawer + lightbox.
- Enter/Space activates buttons; Enter activates anchors.

### Skip link
- Hidden-by-default, visible on focus: `<a href="#main" class="sr-only focus:not-sr-only focus:fixed focus:top-4 focus:left-4 focus:z-50 focus:px-4 focus:py-2 focus:bg-surface focus:text-ink focus:border focus:border-primary focus:rounded-sm">Skip to content</a>`. Sits before the nav.
- `<main id="main">` wraps all 8 sections.

### Touch targets (derived from WIZ-08 precedent)
- All tap targets ≥44×44 px on mobile: hamburger button, nav drawer links (`py-4` on `text-body-lg` ≈ 48px), FAQ summary rows (`min-h-[44px]`), gallery thumbnails (44px minimum — masonry cells are always ≥80px so this is automatic), social icons in footer, all CTAs.

### Reduced motion (LAYT-06, A11Y-06)
- Applies to: nav transition, smooth scroll, drawer translation, menu tab fade, gallery lightbox fade, gallery expand fade, FAQ chevron rotation, hero scroll-down bounce.
- Hero scroll-down bounce uses `motion-safe:animate-bounce` (Tailwind) — zero animation when reduced-motion preferred.
- Smooth scroll falls back to instant via `@media (prefers-reduced-motion: reduce) { html { scroll-behavior: auto } }`.

### Color contrast (A11Y-02)
- All body text on `bg-surface`: `text-ink` (14.5:1, AAA).
- All body text on `bg-ink` (footer bottom): `text-surface/60` min = 7.5:1 (AAA).
- Muted text (`text-ink/60`, `text-ink/70`, `text-ink/80`): verified ≥4.5:1 at each opacity level against cream surface. `text-ink/60` on cream = ~6:1 — passes.
- Accent links (`text-accent`) on cream: 4.7:1 — passes AA for normal text (the minimum standard). Phase 2 MUST NOT reduce opacity on `text-accent` below 100%.
- Hero text on photo: covered by scrim — see Color § Contrast.

### Screen reader semantics
- One `<h1>` (in hero). Sections use `<h2>` for their heading, `<h3>` for subsections (menu categories, package tier names, FAQ category labels).
- `<nav aria-label>`, `<main>`, `<footer>` landmark roles.
- Images: `alt` from schema (non-empty required by schemas for all user-facing images; decorative overlays are `aria-hidden`).
- Form validation errors: N/A Phase 2 (no forms).

### ARIA only when native semantics fall short
- Native `<button>`, `<a>`, `<details>`, `<summary>`, `<nav>`, `<main>`, `<footer>`, `<section>`, `<article>`, `<h1>`-`<h3>`, `<ul>`/`<ol>`/`<li>`, `<blockquote>`, `<cite>` everywhere possible.
- ARIA added for: mobile drawer (`role="dialog"`, `aria-modal`), menu tabs (`role="tab"` / `role="tabpanel"` pattern), gallery lightbox, active nav link (`aria-current="location"`).

## State Machines & Transitions

### Nav state machine

```
┌─────────────┐     hero scroll-out      ┌────────┐
│ transparent │ ────────────────────────▶│ solid  │
└─────────────┘                          └────────┘
       ▲                                     │
       │    hero scroll back into view       │
       └─────────────────────────────────────┘
```

- States: `transparent` (over hero) | `solid` (past hero)
- Trigger: `IntersectionObserver` on `#hero` with `threshold: 0.1`
- When `entry.isIntersecting && entry.intersectionRatio > 0.1`: state = `transparent`
- Else: state = `solid`
- Duration: 300ms ease-out
- Reduced-motion: instant

### Mobile drawer state machine

```
┌────────┐    hamburger tap     ┌──────┐
│ closed │ ────────────────────▶│ open │
└────────┘                      └──────┘
    ▲                              │
    │   tap link/backdrop/X/Esc    │
    └──────────────────────────────┘
```

- States: `closed` | `open`
- Body scroll-lock: applied when `open`, released when `closed`
- Focus: on `open`, focus moves to the close button; on `closed`, focus returns to the hamburger
- Duration: 250ms ease-out translate-x
- Reduced-motion: instant

### Menu tab state machine

```
    ┌────────────┐
    │ proteins   │ ◀──┐
    └────────────┘    │  (default, schema order)
          │           │
          ▼           │
    ┌────────────┐    │
    │ sides      │    │  click or arrow key
    └────────────┘    │
          │           │
          ▼           │
    ┌────────────┐    │
    │ desserts   │────┘
    └────────────┘
```

- States: `proteins` | `sides` | `desserts`
- Default: `proteins`
- Duration: 200ms opacity fade on active panel
- Reduced-motion: instant

### Gallery expand state machine

```
┌──────────┐   "View All" click   ┌──────────┐
│ collapsed│ ────────────────────▶│ expanded │
│ (10 imgs)│                      │ (all)    │
└──────────┘◀─────────────────────└──────────┘
              "Show less" click
```

- States: `collapsed` | `expanded`
- Default: `collapsed` (first 10 images only)
- Duration: 300ms opacity fade-in on newly-shown tiles
- Reduced-motion: instant

### Lightbox state machine

```
┌────────┐  thumbnail click  ┌────────────────┐
│ closed │ ─────────────────▶│ open (index N) │◀──┐
└────────┘                   └────────────────┘   │
    ▲                              │               │ arrow keys / swipe
    │  Esc / X / backdrop click    │               │
    └──────────────────────────────┘               │
                                   ├───────────────┘
                                   │
                                   ▼
                              [lazy-loads next/prev
                               full-size image]
```

- States: `closed` | `open` (with `currentIndex`)
- Trigger: any thumbnail click → `open` with that image's index
- Keyboard: ← → changes index; Esc → `closed`
- Duration: 300ms fade on backdrop + image
- Reduced-motion: 0ms (instant open/close)

### FAQ details state (native `<details>`)

- States: each `<details>` has native `open` attribute (boolean)
- No cross-item state (multiple `<details>` can be open simultaneously — accordion behavior is per-item, not single-open-at-a-time; this matches native web platform default and is the simplest accessible pattern)
- Chevron rotation: 200ms ease-out (CSS only)
- Reduced-motion: no animation on chevron
