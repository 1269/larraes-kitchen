## Critical Anti-Patterns (do NOT repeat these)

- **[ANTI-PATTERN]** Writing Turnstile test keys (`1x00000000000000000000AA`) into `.env` or `.env.production` — SPAM-06 CI gate (`scripts/check-turnstile-keys.sh`) blocks production builds containing them. Test keys MUST be injected via `playwright.config.ts` `webServer.env` (process-scoped only). Never commit test keys to repo env files.
- **[ANTI-PATTERN]** Returning HTTP 200 on webhook handler when the downstream store write fails — CR-04 fix now returns 500 so Resend retries. Do not "swallow" store errors behind a 200.
- **[ANTI-PATTERN]** `Bearer ${cronSecret}` template when `cronSecret` could be undefined — produces `"Bearer undefined"` which any attacker can send. CR-03 fix adds early fail-closed 401 when env is unset. Always guard auth-secret env reads with a fail-closed branch.
