## Shared Patterns (cross-cutting)

### Pattern A: CSS-first design tokens (Tailwind v4 `@theme`)

**Source:** RESEARCH § Code Examples `src/styles/global.css` (lines 466–513)
**Apply to:** `src/styles/global.css` only; every other file consumes tokens via Tailwind utilities (`bg-primary`, `text-ink`, `font-display`, `text-display-xl`).
**Anti-pattern:** No `tailwind.config.js` (forbidden by CLAUDE.md + RESEARCH § Anti-Patterns). No `@astrojs/tailwind` integration (use `@tailwindcss/vite`).

### Pattern B: Zod-schema-as-source-of-truth

**Source:** RESEARCH § Code Examples `src/lib/schemas/packages.ts` (lines 518–539) + `src/content.config.ts` (lines 545–592)
**Apply to:** All 8 files in `src/lib/schemas/` + `src/content.config.ts`.
**Convention:** Each schema module exports both the Zod schema AND the inferred type (`export type PackageData = z.infer<typeof packageSchema>`). Phase 3 will import these same modules for form validation — do not duplicate the shapes.

### Pattern C: `.gitkeep`-only content dirs with `glob("**/*.md")` loader

**Source:** RESEARCH § Code Examples `src/content.config.ts` lines 545–594 + empty-collection note
**Apply to:** All 8 `src/content/<domain>/.gitkeep` placeholders.
**Verification:** `pnpm exec astro sync` must exit 0 on CI. If it errors, fall back to seeding one `_placeholder.md` per collection (RESEARCH § Pitfall 3 + Assumption A2).

### Pattern D: Required status check naming discipline

**Source:** RESEARCH § Pitfall 5 (lines 414–418) + § Code Examples ci.yml job IDs
**Apply to:** `.github/workflows/ci.yml`, `.github/workflows/pr-title.yml`, `scripts/setup-branch-protection.sh`.
**Convention:** Job IDs are lowercase-hyphenated and MUST match the `contexts` array in `setup-branch-protection.sh` byte-for-byte: `typecheck`, `biome-check`, `content-sync`, `image-budget`, `smoke`, `pr-title`.

### Pattern E: Vercel env schema registered day one

**Source:** RESEARCH § Vercel setup recipe (lines 904–921) + § Pitfall 6
**Apply to:** `.env.example` + Vercel Preview + Vercel Production.
**Convention:** Every var listed in `.env.example` is registered in both Vercel environments with value `__placeholder__`. Later phases replace placeholders — they never add new var names without updating `.env.example` and `src/env.d.ts` in the same PR.

### Pattern F: Install-order waves (scaffold sequencing)

**Source:** RESEARCH § Summary (lines 9–13) + § Installation sequence (lines 170–207)
**Apply to:** Wave ordering in the PLAN.
**Convention:** Strict serial order — scaffold wipe → Astro init → framework integrations → Tailwind v4 → shadcn init + Button → tokens → Fontsource + Lovelace → Content Collections → pricing stub → Vitest + Playwright configs → Biome + lefthook → image-budget script → CI workflows → Vercel link → branch protection. Do not collapse waves that share a file.

### Pattern G: Prerender-by-default under `output: 'server'`

**Source:** RESEARCH § Pitfall 4 (lines 408–412)
**Apply to:** `src/pages/index.astro` (and every future page not hitting an Action).
**Convention:** `export const prerender = true;` at top of page. Only Action routes get SSR.

### Pattern H: Commercial-asset license documentation

**Source:** RESEARCH § Pitfall 8 (lines 432–436)
**Apply to:** Lovelace font (and any future commercial asset).
**Convention:** License doc (`public/fonts/LOVELACE-LICENSE.txt` or root `THIRDPARTY.md`) committed alongside the asset; `@font-face` block carries a comment pointing at the license file.
